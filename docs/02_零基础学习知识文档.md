## Phase 2：PyTorch 基础

这一章是重中之重。PyTorch 是整个项目的基石，我们会从最核心的概念讲起。

### 2.1 Tensor：PyTorch 的"数字容器"

Tensor 就是 PyTorch 版的"数组"。你可以把它想象成一个装数字的多维盒子。

```python
import torch

# 从 Python 列表创建 Tensor
a = torch.tensor([1, 2, 3])
print(a)        # tensor([1, 2, 3])
print(a.shape)  # torch.Size([3])

# 二维 Tensor（矩阵）
b = torch.tensor([[1, 2], [3, 4]])
print(b.shape)  # torch.Size([2, 2])

# 全零矩阵
c = torch.zeros(3, 4)   # 3行4列，全是0
# 全一矩阵
d = torch.ones(2, 5)    # 2行5列，全是1
# 随机矩阵
e = torch.randn(2, 3)   # 2行3列，从正态分布随机采样
f = torch.rand(2, 3)   # 从区间 [0, 1) 均匀分布采样
```

**Tensor 的"形状"（shape）** 非常重要。在深度学习中，数据都是以 Tensor 的形式流动的。图片是 `(B, C, H, W)`，其中 B 是批大小，C 是通道数（RGB=3），H 是高，W 是宽。

**把 Tensor 搬到 GPU 上：**

```python
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
# 一般可以直接把 ”cuda“ 后面东西删掉
a = torch.tensor([1, 2, 3]).to(device)  # 搬到 GPU
# 也可以写成两句话
a = torch.tensor([1, 2, 3])
a = a.to(device)

print(a.device)  # cuda:0 即第0块cuda
```

**数据类型：**

```python
a = torch.tensor([1, 2, 3], dtype=torch.float32)  # 浮点数 去掉32也可以
b = torch.tensor([1, 2, 3], dtype=torch.long)      # 整数（标签用）
c = torch.tensor([1, 2, 3], dtype=torch.int)   # 输出会变为tensor([1, 2, 3], dtype=torch.int32) 也就是后面多了dtype, 但其他数据类型不会有
```

### 2.2 Autograd：自动求导

神经网络的训练核心是"反向传播"，也就是计算每个参数对 Loss 的梯度，然后更新参数。PyTorch 的 autograd 会自动帮你算梯度。

```python
x = torch.tensor([2.0], requires_grad=True)  # 告诉 PyTorch：我要算这个变量的梯度，不加的话默认不自动求导，torch.randn()同样
y = x ** 2 + 3 * x + 1                        # y = x^2 + 3x + 1
y.backward()                                   # 反向传播，自动算 dy/dx
print(x.grad)                                  # tensor([7.]) 因为 dy/dx = 2x+3 = 7
```

**核心概念：**

- `requires_grad=True`：告诉 PyTorch 这个变量需要算梯度
- `backward()`：从当前 Tensor 开始反向传播，计算所有 `requires_grad=True` 的变量的梯度
- `.grad`：存着算好的梯度值
- `.grad_fn`：记录这个 Tensor 是怎么算出来的（比如 `y.grad_fn` 会告诉你 y 是由乘法算出来的）

**需要注意一些东西：**

```Python
x = torch.tensor([2.0])
x.require_grad_(True)  # 可以后面补上需要自动求导
y = x ** 2 + 3 * x + 1
z = y * 2
z.backward()  # 这里求z关于x的偏导，此外不能再有y.backward()，原因是backward不能跑两次同样的计算图 (即计算图不能有重叠)
print(x.grad)  # 正常输出
print(y.grad)  # 输出None 两个原因：1.没有要求其自动求导 2.其不是叶子张量，而Torch是不会保存非叶子张量的求导结果的，所以即便给 y 加上了 y.require_grad_(Ture) 依旧会输出None
# 如果真的需要非叶子张量的求导结果，可以使用:
y.retain_grad()
# 另外还需要注意一个点，就是.to(device)也会使张量变成非叶子张量
```

**计算图的概念：**

PyTorch 在背后维护了一张"计算图"。每次你做运算，PyTorch 就记下"谁是谁的输入"。调用 `backward()` 时，PyTorch 沿着这张图从后往前算梯度。

```python
a = torch.tensor([2.0], requires_grad=True)
b = torch.tensor([3.0], requires_grad=True)
c = a * b          # c.grad_fn = <MulBackward0>
d = c + 1          # d.grad_fn = <AddBackward0>
e = torch.sin(d)   # e.grad_fn = <SinBackward0>
# 操作名后面跟着的数字通常代表 该运算对应计算图中输入的索引（Input Index）
# 一般输出是 <MulBackward0 object at 地址码>
e.backward()
print(a.grad)  # de/da = cos(a*b+1) * b = cos(7) * 3 ≈ 2.27
print(b.grad)  # de/db = cos(a*b+1) * a = cos(7) * 2 ≈ 1.51
```

**为什么需要清零梯度？**

```python
optimizer.zero_grad()  # 把梯度清零
loss.backward()        # 算新的梯度
optimizer.step()       # 更新参数
```

如果不清零，每次 `backward()` 算出的梯度会累加到 `.grad` 上。这在某些场景有用（梯度累积），但大多数时候我们需要的是"当前 batch 的梯度"。

实例代码：

```python
import torch
import torch.optim as optim  # 需要先导入optim
import torch.nn as nn  # 导入nn

# 1. 定义模型和优化器
model = nn.Linear(10, 1)
# 传入 model.parameters() 告诉优化器要更新哪些权重
optimizer = optim.SGD(model.parameters(), lr=0.01, momentum=0.9)

# 在训练循环中：
for input, target in dataset:
    # --- 第一步：清空梯度 ---
    # 必须清空，否则 PyTorch 会累加之前的梯度
    optimizer.zero_grad()
    
    # --- 第二步：前向传播 ---
    output = model(input)
    loss = criterion(output, target)
    
    # --- 第三步：反向传播 ---
    # 计算每个 Parameter 的 .grad 值
    loss.backward()
    
    # --- 第四步：更新参数 ---
    # 根据权重 = 权重 - 学习率 * 梯度 (以SGD为例) 进行更新
    optimizer.step()
```

在训练中，模型的参数Parameter（权重）默认 `requires_grad=True`，输入数据默认 `requires_grad=False`。

### 2.3 nn.Module：所有模型的基类

PyTorch 里所有的神经网络层都继承自 `nn.Module`。你自己写的模型也要继承它。

```python
import torch.nn as nn

class MyFirstModel(nn.Module):
    def __init__(self):
        super().__init__()           # 必须调用父类的 __init__
        self.fc = nn.Linear(10, 5)   # 一个全连接层：输入10维，输出5维
    
    def forward(self, x):
        return self.fc(x)            # 前向传播：输入 -> 输出

model = MyFirstModel()
x = torch.randn(3, 10)              # 3个样本，每个10维
output = model(x)                    # 调用 forward 可以写成model.forward(x)
print(output.shape)                  # torch.Size([3, 5])
```

**两个必须实现的方法：**

- `__init__`：定义模型有哪些层（"零件"）
- `forward`：定义数据怎么流过这些层（"组装"）

### 2.4 Tensor 的索引和切片

Tensor 的索引和 Python 列表很像，但功能更强大。

```python
x = torch.randn(4, 3, 512, 512)  # (batch, channel, height, width)

# 取第一张图片
img0 = x[0]          # shape: (3, 512, 512)

# 取第一张图片的红色通道
red_channel = x[0, 0]  # shape: (512, 512)

# 取所有图片的绿色通道
green_channels = x[:, 1]  # shape: (4, 512, 512)

# 取前两张图片
batch = x[:2]  # shape: (2, 3, 512, 512)

# 用布尔索引筛选
mask = x > 0
positive_values = x[mask]  # 将满足mask条件的元素抽出成成一维张量
```

**广播机制：**

不同形状的 Tensor 做运算时，PyTorch 会自动"广播"到相同的形状。

```python
a = torch.randn(4, 3, 512, 512)
b = torch.randn(512, 512)  # 会自动广播到 (1, 1, 512, 512) 再和 a 运算
c = a + b  # 没问题，b 被广播了
```

### 2.5 常用神经网络层

#### nn.Linear（全连接层）

```python
# 输入特征数 384，输出特征数 256
layer = nn.Linear(384, 256)
x = torch.randn(2, 384)    # 2个样本
y = layer(x)                # shape: (2, 256)
```

做的事情：`y = xW^T + b`，其中 W 是权重矩阵，b 是偏置。

#### nn.Conv2d（二维卷积层）

```python
# 输入通道3（RGB），输出通道64，卷积核大小3x3
layer = nn.Conv2d(in_channels=3, out_channels=64, kernel_size=3, padding=1)
x = torch.randn(2, 3, 512, 512)  # 2张RGB图片，512x512
y = layer(x)                       # shape: (2, 64, 512, 512)
```

卷积层在图片上滑动一个小窗口（kernel），提取局部特征。`padding=1` 表示在图片边缘补一圈0，让输出尺寸不变（填充）。

#### nn.BatchNorm2d（批归一化）

```python
layer = nn.BatchNorm2d(64)  # 输入通道数64
```

对每个通道做归一化：减去均值，除以标准差。让训练更稳定，收敛更快。

#### nn.Dropout2d（随机丢弃）

```python
layer = nn.Dropout2d(p=0.1)  # 10%的概率丢弃
```

训练时随机把一些通道整张图都置为0，防止过拟合。测试时自动关闭。

### 2.5 nn.Sequential：顺序容器

当你有一串层要依次执行时，用 `nn.Sequential` 可以省去手写 `forward`。

```python
block = nn.Sequential(
    nn.Conv2d(64, 128, kernel_size=3, padding=1),
    nn.BatchNorm2d(128),
    nn.ReLU(inplace=True),  # inplace=True 表示直接在原地址更改
    nn.Dropout2d(p=0.1)
)
x = torch.randn(2, 64, 32, 32)
y = block(x)  # 自动依次执行所有层
```

### 2.6 nn.Embedding：可学习的查找表

这是 EoMT 的核心组件之一。Embedding 就像一个"字典"，每个索引对应一个向量。

```python
# 创建10个向量，每个384维
embed = nn.Embedding(num_embeddings=10, embedding_dim=384) # 前者是创建的个数，后者是维度，一开始这里都是随机数，随着学习会不断成为单词真正的意思
indices = torch.tensor([0, 3, 7])  # 要查的索引
# 由于索引必须是整数，所以我们一般会用torch.LongTensor() 即产生一个64位int型张量
vectors = embed(indices)            # shape: (3, 384) 这是默认batch=1的情况，多批次时shape为(2, 3, 384)
```

在 EoMT 中，我们用 `nn.Embedding(num_q, embed_dim)` 来创建一组"可学习的查询向量"。这些向量会在训练中不断调整，学会关注图片的不同区域。

### 2.7 nn.ModuleList：模块列表

当你有一堆模块要存起来，而且每个模块的参数都需要被优化器看到时，用 `nn.ModuleList`。

```python
layers = nn.ModuleList([
    nn.Linear(384, 384) for _ in range(6)
])
x = torch.randn(2, 384)
for layer in layers:
    x = layer(x)
```

注意：用普通的 Python 列表存模块，PyTorch 看不到里面的参数，优化器不会更新它们。

### 2.8 nn.GELU：激活函数

GELU（Gaussian Error Linear Unit）是 Transformer 里常用的激活函数，比 ReLU 更平滑。

```python
gelu = nn.GELU()
x = torch.tensor([-2.0, -1.0, 0.0, 1.0, 2.0])
y = gelu(x)
```

ReLU 在负数区域直接截断为0，GELU 则给负数一个很小的非零值，让梯度流动更顺畅。

### 2.9 损失函数：CrossEntropyLoss

分类任务的标准损失函数。

```python
criterion = nn.CrossEntropyLoss(ignore_index=255)
```

- 输入：模型预测 `(B, C, H, W)`，真实标签 `(B, H, W)`
- 对每个像素，比较预测的类别和真实类别
- `ignore_index=255`：忽略标签值为255的像素（VOC 数据集的边缘区域）

### 2.10 优化器：AdamW

```python
import torch.optim as optim

optimizer = optim.AdamW(model.parameters(), lr=1e-3)
```

- `model.parameters()`：模型所有需要训练的参数
- `lr`：学习率，控制每次更新的步长
- AdamW 是 Adam 的改进版，把权重衰减（weight decay）从梯度里分离出来，效果更好

**Adam vs AdamW：**

Adam 的权重衰减和梯度更新混在一起，AdamW 把它们分开处理。实践中 AdamW 通常效果更好，尤其是 Transformer 模型。

### 2.11 学习率调度

学习率不是一成不变的。常见策略是"多项式衰减"：训练初期大步走，后期小步微调。

```python
# 多项式衰减：lr = base_lr * (1 - iter / total_iters) ^ power
# power=0.9 是 EoMT 常用的值
```

### 2.12 DataLoader：数据加载器

DataLoader就是按照某种要求从庞大的数据集里批量加载数据，让模型按批处理数据。

```python
from torch.utils.data import DataLoader

loader = DataLoader(dataset, batch_size=4, shuffle=True, drop_last=True)
for images, masks in loader:
    # images shape: (4, 3, 512, 512)
    # masks shape: (4, 512, 512)
    pass
```

- `batch_size`：每次取多少张图片
- `shuffle=True`：打乱顺序，让模型看到不同的数据顺序
- `drop_last=True`：如果最后一批不够 batch_size，丢掉它

### 2.13 Dataset：自定义数据集

```python
from torch.utils.data import Dataset

class MyDataset(Dataset):
    def __init__(self):
        # 初始化：读文件列表，做预处理
        self.data = list(range(100)) # range(start,stop,step)作用是生成一个等差整数序列
    
    def __len__(self):
        # 返回数据总量
        return len(self.data)
    
    def __getitem__(self, idx):
        # 返回第 idx 个样本
        x = torch.tensor(self.data[idx], dtype=torch.float32)
        y = torch.tensor(self.data[idx] % 2, dtype=torch.long)  # 标签(奇偶性)
        return x, y
```

**必须实现三个方法：**

- `__init__`：初始化，读数据
- `__len__`：返回数据总量
- `__getitem__`：返回第 idx 个样本（通常是 (输入, 标签) 对）

### 2.14 模型保存与加载

训练好的模型要保存下来，下次直接用。

```python
# 保存模型权重（推荐方式）
torch.save(model.state_dict(), "my_model_weights.pth")

# 加载模型权重
model = MyModel()
model.load_state_dict(torch.load("my_model_weights.pth"))
model.eval()  # 切换到推理模式
```

`state_dict` 是模型所有参数的字典。只保存权重不保存模型结构，所以加载前要先创建模型实例。

### 2.15 训练模式与评估模式

```python
model.train()  # 训练模式：Dropout 生效，BatchNorm 用 batch 统计量
model.eval()   # 评估模式：Dropout 关闭，BatchNorm 用全局统计量
```

忘记切换模式是个很常见的 bug。训练时 Dropout 随机丢弃神经元，评估时如果还丢弃，结果就不稳定了。

`.pth` 文件是 PyTorch 用来保存模型**各种参数**的二进制文件，主要存储的是 **`state_dict`（状态字典）**。 它本质上是一个 Python 字典，把模型每一层的名称（比如 `layer1.weight`）映射到对应的权重张量（Tensor）上。

### 2.16 梯度累积

当显存不够、batch_size 设得很小时，可以用梯度累积来模拟更大的 batch：

也就是每次还是加载很小的batch，但积累到一定数量后再统一经过optimizer.

```python
accumulation_steps = 4  # 累积 4 次再更新
optimizer.zero_grad()

for i, (images, masks) in enumerate(loader):
    outputs = model(images)
    loss = criterion(outputs, masks)  # 计算loss
    loss = loss / accumulation_steps  # 平均一下
    loss.backward()
    
    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

这样虽然每次只喂 2 张图，但累积 4 次的效果相当于 batch_size=8。

`enumerate()` 是 Python 中一个非常优雅的内置函数。它的核心作用是：**在遍历一个序列（如列表、元组或字符串）时，同时获得元素的“索引（Index）”和“值（Value）”**

### 2.17 随机种子

为了让实验结果可复现，设置随机种子：

```python
import random
import numpy as np
import torch

def set_seed(seed=42):
    random.seed(seed)
    np.random.seed(seed)
    torch.manual_seed(seed)
    torch.cuda.manual_seed_all(seed)
    torch.backends.cudnn.deterministic = True
```

这样每次跑出来的结果都一样，方便调试和对比实验。

### 2.18 完整的训练循环

把上面所有概念串起来：

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, Dataset

# 1. 数据
class ToyDataset(Dataset):
    def __len__(self):
        return 100
    def __getitem__(self, idx):
        return torch.randn(10), torch.randint(0, 3, ())

dataset = ToyDataset()
loader = DataLoader(dataset, batch_size=8, shuffle=True)

# 2. 模型
model = nn.Sequential(
    nn.Linear(10, 20),
    nn.GELU(),
    nn.Linear(20, 3)
)

# 3. 损失和优化器
criterion = nn.CrossEntropyLoss()
optimizer = optim.AdamW(model.parameters(), lr=1e-3)

# 4. 训练
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model.to(device)

for epoch in range(5):
    for x, y in loader:
        x, y = x.to(device), y.to(device)
        
        pred = model(x)
        loss = criterion(pred, y)
        
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
    
    print(f"Epoch {epoch+1}, Loss: {loss.item():.4f}")
```

## Phase 3：视觉 Transformer（ViT）基础

### 3.1 为什么需要 Transformer

在进入 ViT 之前，先想想传统方法有什么问题。

**CNN 的局限：**

CNN 的核心操作是卷积，一个小窗口在图片上滑动。这个窗口通常只有 3x3 或 5x5。所以 CNN 的"视野"很窄，每个卷积核只能看到周围几个像素。

要看到更大的范围，CNN 需要堆很多层。底层看到小范围，中层看到中等范围，高层才能看到全局。这个过程需要很多层，而且信息在传递过程中可能会丢失。

**Transformer 的优势：**

Self-Attention 让每个位置直接和所有其他位置通信。第一层就能看到整张图。这就像：

- CNN：你通过一个纸筒看一幅画，只能看到一个小圆点。你得移动纸筒很多次，然后在脑子里拼出整幅画。
- Transformer：你直接站在画前面，一眼看到整幅画的所有细节。

对于语义分割来说，全局视野特别重要。要判断一个像素是"车"还是"路"，你需要知道它周围更大范围的情况。Transformer 天生就擅长这个。

### 3.2 从 CNN 到 Transformer

在 Transformer 出现之前，计算机视觉全靠 CNN（卷积神经网络）。CNN 有个天然限制：**局部感受野**。一个卷积核只能看到它周围的一小块区域，想看远处的东西得堆很多层。

Transformer 不一样。它的 **Self-Attention** 机制让每个位置都能"看到"所有其他位置。一张 512x512 的图片，CNN 第一层只能看 3x3 的范围，Transformer 第一层就能看整张图。

**为什么这很重要？**

语义分割需要理解"全局上下文"。比如，要判断一个像素是"车"还是"路"，你得知道它周围更大范围的情况。Transformer 天生擅长这个。

### 3.2 Patch Embedding：把图片切成单词

Transformer 本来是为文本设计的，处理的是"单词序列"。图片怎么变成"序列"呢？

答案是：把图片切成小块（patch），每个小块当成一个"单词"。

```python
import torch
import torch.nn as nn

class PatchEmbed(nn.Module):
    def __init__(self, img_size=512, patch_size=16, in_chans=3, embed_dim=384):
        super().__init__()
        self.proj = nn.Conv2d(in_chans, embed_dim, kernel_size=patch_size, stride=patch_size)
    
    def forward(self, x):
        # x shape: (B, 3, 512, 512)
        x = self.proj(x)  # (B, 384, 32, 32)  -- 512/16=32
        x = x.flatten(2)  # (B, 384, 1024)     -- 32*32=1024个patch
        x = x.transpose(1, 2)  # (B, 1024, 384) -- 1024个token，每个384维
        return x
```

**关键数字：**

- 图片 512x512，patch 大小 16x16
- 一共 `(512/16) * (512/16) = 1024` 个 patch
- 每个 patch 被线性投影成 384 维的向量（称为一个 token）
- 输出形状：`(B, 1024, 384)` ，就像 1024 个"单词"，每个 384 维

### 3.3 Position Embedding：给 token 标位置

Self-Attention 有个特点：**它不在乎顺序**。把 token 的顺序打乱，结果一样。这对文本来说是个问题（"我打你"和"你打我"意思完全不同），对图片也一样（左上角和右下角的 patch 应该不同）。

解决办法：给每个 token 加上一个"位置编码"。

```python
# 可学习的位置编码
num_patches = 1024
embed_dim = 384
pos_embed = nn.Parameter(torch.randn(1, num_patches, embed_dim))

# 加到 patch token 上
x = patch_embed(image)      # (B, 1024, 384)
x = x + pos_embed           # (B, 1024, 384) -- 每个位置加上对应的编码，这个位置编码只和位置有关，和批次无关，所以直接使用广播
```

`nn.Parameter` 告诉 PyTorch：这是一个可学习的参数，需要算梯度。位置编码会在训练中自动调整。

### 3.4 Multi-Head Self-Attention（MHSA）

这是 Transformer 的核心。先理解 **Single-Head Self-Attention**：

**Q、K、V 是什么？**

- **Q（Query，查询）**：当前 token 想问的问题
- **K（Key，键）**：其他 token 的"标签"
- **V（Value，值）**：其他 token 的"内容"

流程：每个 token 发出 Q，所有 token 的 K 和 Q 做匹配，匹配分数高的 V 被更多地采纳。

**注意力权重计算公式：**

```
Attention(Q, K, V) = softmax(QK^T / sqrt(d)) V
```

- `QK^T`：Q 和 K 的点积，衡量相似度
- `除以 sqrt(d)`：防止点积太大导致 softmax 过于尖锐
- `softmax`：把分数变成概率（和为1）
- `乘以 V`：按概率加权求和

**Multi-Head 就是做多次 Self-Attention：**

```python
# 8个头，每个头64维，总共512维
mhsa = nn.MultiheadAttention(embed_dim=384, num_heads=6, batch_first=True)
```

多个头可以关注不同的方面。一个头关注颜色，另一个头关注形状，等等。

### 3.5 MLP 层

Self-Attention 之后，每个 token 独立经过一个 MLP（多层感知机）：

```python
mlp = nn.Sequential(
    nn.Linear(384, 384 * 4),  # 先扩大4倍
    nn.GELU(),                 # 激活
    nn.Linear(384 * 4, 384),   # 再缩回原尺寸
)
```

"先扩大再缩小"的设计让模型有更多空间做特征变换。

### 3.6 Layer Normalization

BatchNorm 是对"同一个通道的所有像素"做归一化。LayerNorm 是对"同一个 token 的所有维度"做归一化。

```python
layer_norm = nn.LayerNorm(384)
x = torch.randn(2, 1024, 384)
x = layer_norm(x)  # 对每个 token 独立做归一化
```

Transformer 用 LayerNorm 而不是 BatchNorm，因为 LayerNorm 不依赖 batch 大小，而且对变长序列更友好。

### 3.7 Residual Connection（残差连接）

Transformer 的每个子层（Self-Attention 和 MLP）外面都包了一层残差连接：

```
x = x + sublayer(norm(x))
```

```python
# 一个完整的 Transformer Block
class TransformerBlock(nn.Module):
    def __init__(self, dim=384, num_heads=6):
        super().__init__()
        self.norm1 = nn.LayerNorm(dim)
        self.attn = nn.MultiheadAttention(dim, num_heads, batch_first=True)
        self.norm2 = nn.LayerNorm(dim)
        self.mlp = nn.Sequential(
            nn.Linear(dim, dim * 4),
            nn.GELU(),
            nn.Linear(dim * 4, dim),
        )
    
    def forward(self, x):
        # Self-Attention + 残差
        x = x + self.attn(self.norm1(x), self.norm1(x), self.norm1(x))[0]
        # MLP + 残差
        x = x + self.mlp(self.norm2(x))
        return x
```

残差连接解决了深层网络的梯度消失问题，让信息能直接流过整个网络。

### 3.8 ViT 整体结构

把上面所有组件拼起来，就是一个完整的 ViT：

```
输入图片 (B, 3, 512, 512)
    |
Patch Embedding (B, 1024, 384)
    |
+ Position Embedding
    |
[Transformer Block × N]  ← 重复多次
    |  每个 Block 包含：
    |    Norm -> MHSA -> Residual
    |    Norm -> MLP -> Residual
    |
Layer Normalization
    |
输出特征 (B, 1024, 384)
```

**为什么 ViT 需要大量数据训练？**

ViT 比 CNN 更"贪吃"。CNN 有归纳偏置（inductive bias），它天生假设"相邻像素更相关"，所以用小数据也能学。ViT 没有这个假设，它需要从数据中自己学会"相邻像素更相关"这件事。

所以 ViT 通常需要在大数据集（比如 ImageNet-21k、JFT-300M）上预训练，然后再在小数据集上微调。DINOv3 在 14 亿张图片上做了自监督训练，已经学好了"怎么看图"。

**ViT 的三种常见尺寸：**

| 型号 | 参数量 | Patch 大小 | 隐藏维度 | 头数 | 层数 |
|------|--------|-----------|---------|------|------|
| ViT-Small | 22M | 16 | 384 | 6 | 12 |
| ViT-Base | 86M | 16 | 768 | 12 | 12 |
| ViT-Large | 307M | 14 | 1024 | 16 | 24 |

我们用的 DINOv3 ViT-Small 是最轻量的版本，参数量只有 2200 万，适合在消费级显卡上跑。

### 3.9 Self-Attention 的数学直觉

很多人看到公式就头疼，我们来拆解一下：

```
Attention(Q, K, V) = softmax(QK^T / sqrt(d)) V
```

**第一步：QK^T 算相似度**

假设有 3 个 token，每个 4 维。Q 是 (3,4)，K^T 是 (4,3)，乘起来是 (3,3)。这个 3x3 矩阵的 (i,j) 位置表示"第 i 个 token 和第 j 个 token 有多像"。

**第二步：除以 sqrt(d) 防饱和**

d 是每个头的维度。如果不除，点积值可能很大（比如 100），softmax 之后几乎变成 one-hot（一个 1，其他接近 0），梯度就消失了。除以 sqrt(d) 让值保持在合理范围。

**第三步：softmax 归一化**

把每行的分数变成概率，和为 1。这样每个 token 从其他 token "吸收"信息时，权重加起来正好是 1。

**第四步：乘以 V 加权求和**

V 是每个 token 的"内容"。用 softmax 算出的权重对 V 做加权平均，得到每个 token 的新表示。

**一个具体的例子：**

假设 token A 是"猫头"，token B 是"猫身体"，token C 是"背景"。Self-Attention 之后，token A 的新表示会包含来自 token B 的信息（因为它们是同一只猫），而来自 token C 的信息很少（因为背景和猫不相关）。

### 3.9 用 timm 加载 DINOv3

我们不用自己写 ViT，直接用 `timm` 库加载预训练好的 DINOv3：

```python
import timm

# 加载 DINOv3 ViT-Small，patch size 16，输入 384x384
model = timm.create_model(
    'vit_small_patch16_dinov3.lvd1689m',
    pretrained=True,
    features_only=True  # 返回 2D 特征图，而不是 token 序列
)
```

**`features_only=True` 的作用：**

- 默认 ViT 输出 token 序列 `(B, N, C)`
- 开启后输出 2D 特征图 `(B, C, H, W)`，方便接卷积层做分割

### 3.10 冻结预训练权重

DINOv3 已经在海量图片上训练好了，我们不想破坏它的知识，只想在它上面加一个"小头"做分割。

```python
for param in model.parameters():
    param.requires_grad = False  # 冻结所有参数
```

冻结后，反向传播不会更新这些参数，节省显存和计算量。

### 3.11 DINOv3 的训练方式

DINOv3 不是普通的 ViT，它用了自监督学习训练。什么意思呢？

普通 ViT 训练需要人工标注的数据（比如 ImageNet 的 1000 类标签）。DINOv3 不需要标签，它自己和自己玩。

**自监督学习的思路：**

1. 对同一张图片做两种不同的数据增强（裁剪、变色、旋转等）
2. 让模型对这两种增强版本提取的特征尽可能相似
3. 这样模型就学会了"不管图片怎么变，猫还是猫"

DINOv3 在 14 亿张图片上做了自监督训练。所以它的特征非常通用，适合各种下游任务（分类、检测、分割）。

**为什么这对我们很重要？**

因为 DINOv3 的特征是"通用的"，它学到的不是"这个物体是猫"这种具体知识，而是"这个区域的纹理、形状、颜色特征是什么"。这些通用特征可以直接用来做语义分割。

### 3.12 DINOv3 的特殊设计

DINOv3 在标准 ViT 基础上做了几个改动：

- **Register tokens**：额外添加的一些 token，用来"寄存"局部信息，防止 patch token 丢失细节
- **[CLS] token**：一个特殊的 token，用来做全局分类（我们做分割时不用它）
- **Prefix tokens**：在 patch token 前面插入的一些 token

在 EoMT 中，我们会在这些 token 后面再插入我们自己的"查询 token"。

## Phase 4：语义分割与评估指标

### 4.1 什么是语义分割

语义分割就是"给图片的每个像素分类"。比如一张街景图，把天空像素标为"天空"，把路像素标为"路"，把人像素标为"人"。

和"目标检测"（画框框出物体）不同，语义分割要精确到每个像素。

### 4.2 PASCAL VOC 2012 数据集

PASCAL VOC 2012 是语义分割领域最经典的数据集之一：

- **20 个物体类别** + **1 个背景类别** = **21 类**
- 类别包括：飞机、自行车、鸟、船、瓶子、公交车、小汽车、猫、椅子、牛、餐桌、狗、马、摩托车、人、盆栽、羊、沙发、火车、电视
- 训练集约 1400 张图片，验证集约 1400 张

**标签处理：**

每张图片对应一张"标签图"，标签图上每个像素的值就是类别编号（0 到 20）。但 VOC 的标签图边缘有一圈"未标注"区域，这些像素的值是 255。

```python
# 在损失函数中忽略 255
criterion = nn.CrossEntropyLoss(ignore_index=255)
```

### 4.3 mIoU：最重要的指标

**mIoU = Mean Intersection over Union（平均交并比）**

先理解 IoU（单个类别）：

```
IoU = TP / (TP + FP + FN)
```

- **TP（True Positive）**：模型说是"猫"，实际也是"猫"的像素数
- **FP（False Positive）**：模型说是"猫"，实际不是"猫"的像素数
- **FN（False Negative）**：模型没说"猫"，实际是"猫"的像素数

IoU 衡量的是"模型预测的这个类别"和"真实这个类别"的重叠程度。1 表示完美重合，0 表示完全不沾边。

**mIoU 就是所有类别 IoU 的平均值。**

### 4.4 Pixel Accuracy（像素准确率）

```
Pixel Accuracy = 预测正确的像素数 / 总像素数
```

这个指标虽然直观，但有个问题：如果背景占了 90% 的像素，模型什么都不做只预测"背景"，准确率也有 90%。所以 mIoU 才是更可靠的指标。

### 4.5 为什么 mIoU 比 Pixel Accuracy 更好

用一个例子说明。假设一张图片有 100 个像素：

- 90 个像素是背景（类别 0）
- 10 个像素是猫（类别 1）

模型 A 把所有像素都预测为背景：

- Pixel Accuracy = 90/100 = 90%
- mIoU = (IoU_背景 + IoU_猫) / 2
  - IoU_背景 = 90 / (90 + 0 + 10) = 0.9
  - IoU_猫 = 0 / (0 + 0 + 10) = 0
  - mIoU = (0.9 + 0) / 2 = 45%

模型 B 预测对了 80 个背景和 8 个猫：

- Pixel Accuracy = 88/100 = 88%
- mIoU = (IoU_背景 + IoU_猫) / 2
  - IoU_背景 = 80 / (80 + 10 + 2) = 0.87
  - IoU_猫 = 8 / (8 + 2 + 2) = 0.67
  - mIoU = (0.87 + 0.67) / 2 = 77%

模型 A 的 Pixel Accuracy 更高（90% vs 88%），但 mIoU 更低（45% vs 77%）。mIoU 更能反映模型"真的学会了分割猫"，而 Pixel Accuracy 被大量背景像素"稀释"了。

这就是为什么语义分割领域主要看 mIoU。

### 4.6 fast_hist：高效计算混淆矩阵

```python
import numpy as np

def fast_hist(a, b, n):
    # a: 真实标签, b: 预测标签, n: 类别数
    k = (a >= 0) & (a < n)           # 过滤掉 255
    return np.bincount(
        n * a[k].astype(int) + b[k],  # 把二维索引压成一维
        minlength=n ** 2
    ).reshape(n, n)                   # 变回 n x n 的混淆矩阵
```

**原理：** 混淆矩阵是 n x n 的表格，`hist[i][j]` 表示"真实类别 i 被预测为类别 j"的像素数。`n * i + j` 把二维索引变成一维，`bincount` 统计每个一维索引出现的次数，最后 `reshape` 回来。

从混淆矩阵算 IoU：

```python
hist = np.zeros((21, 21))  # 21x21 的混淆矩阵
# ... 遍历所有图片，累加 hist ...

# 每个类别的 IoU
iou_per_class = np.diag(hist) / (hist.sum(axis=1) + hist.sum(axis=0) - np.diag(hist))
# 平均 IoU（忽略 NaN）
miou = np.nanmean(iou_per_class)
```

## Phase 5：EoMT 论文核心概念精讲

这是最关键的一章。EoMT（Encoder-only Mask Transformer）的核心思想是：**只用 ViT 编码器，不要解码器，也能做语义分割。**

### 5.1 为什么 EoMT 不需要解码器

传统语义分割模型（比如 PSPNet）的结构是：

```
编码器（提取特征） -> 解码器（恢复分辨率） -> 分类
```

编码器把图片压缩成特征，解码器把特征恢复成原图大小的预测。解码器通常很复杂，比如 PSPNet 的金字塔池化模块、DeepLab 的空洞卷积等。

EoMT 的想法很直接：**如果编码器足够强（比如 DINOv3），它自己就能做分割，不需要额外的解码器。**

怎么做呢？在 ViT 的中间层插入一些"可学习的查询向量"，让这些查询向量学会关注图片的不同区域。这些查询向量就像一群"小助手"，每个助手负责图片的一个区域，帮编码器完成分割任务。

**EoMT 和 PSPNet 的对比：**

| 对比项 | PSPNet | EoMT |
|--------|--------|------|
| 骨干网络 | DINOv3（冻结） | DINOv3（可微调） |
| 额外模块 | 金字塔池化 + 卷积头 | 可学习查询 + 几层 Transformer |
| 解码器 | 有（PPM + 分类器） | 无（编码器自己搞定） |
| 核心思想 | 多尺度池化聚合上下文 | 可学习查询提取区域信息 |

### 5.2 Learnable Queries（可学习查询）

这是 EoMT 最核心的创新。

```python
import torch.nn as nn

num_queries = 100       # 100 个查询
embed_dim = 384         # 和 ViT 的维度一致

queries = nn.Embedding(num_queries, embed_dim)
# queries.weight.shape: (100, 384)
```

**这些查询是什么？**

你可以把每个查询想象成一个"探针"。训练开始时，100 个探针都是随机初始化的。训练过程中，它们会学会：

- 探针 A 负责关注"天空"区域
- 探针 B 负责关注"人"的区域
- 探针 C 负责关注"车"的区域
- 等等

每个探针最终会对应一个"分割区域"。

### 5.3 查询插入位置

查询不是在开头就插入的，而是在 ViT 的中间层插入。

```
ViT 有 L 层 Transformer Block
    |
前 L1 层：纯特征提取（让 DINOv3 先充分理解图片）
    |
插入查询 token（和 patch token 拼接在一起）
    |
后 L2 层：查询和 patch 一起做 Self-Attention
    |
输出
```

**为什么不在开头插入？**

因为一开始图片特征还没提取好，查询不知道该关注什么。先让 DINOv3 处理几层，等特征丰富后再插入查询，查询就能"站在巨人的肩膀上"工作。

**插入后的 Self-Attention 同时做三件事：**

1. **Query <-> Query**：查询之间互相通信，协调各自负责的区域（避免两个查询抢同一个区域）
2. **Query <-> Patch**：查询从 patch token 提取信息（"这个区域有什么特征？"）
3. **Patch <-> Patch**：patch token 之间继续做特征提取

这三件事在同一个 MHSA 里完成，非常高效。

### 5.4 Class Head 和 Mask Head

每个查询需要输出两样东西：

1. **它属于哪个类别**（Class Head）
2. **它对应的区域在哪里**（Mask Head）

```python
class EoMTHead(nn.Module):
    def __init__(self, embed_dim=384, num_classes=21, num_queries=100):
        super().__init__()
        self.class_head = nn.Linear(embed_dim, num_classes)  # 分类
        self.mask_head = nn.Linear(embed_dim, embed_dim)     # 生成 mask 嵌入
    
    def forward(self, query_features):
        # query_features: (B, num_queries, embed_dim)
        class_logits = self.class_head(query_features)  # (B, num_queries, 21)
        mask_embed = self.mask_head(query_features)     # (B, num_queries, embed_dim)
        return class_logits, mask_embed
```

### 5.5 Mask 预测：点积操作

Mask 预测不是直接输出一个 mask，而是通过"点积"来生成：

```python
# patch_features: (B, H*W, embed_dim)  -- 所有 patch token 的特征
# mask_embed: (B, num_queries, embed_dim)  -- 每个查询的 mask 嵌入
# mask_logits: (B, num_queries, H*W)

mask_logits = torch.matmul(mask_embed, patch_features.transpose(1, 2))
```

**直觉理解：**

- 每个查询有一个"模板向量"（mask_embed）
- 每个 patch 有自己的特征向量
- 点积衡量"这个 patch 和这个查询的模板有多匹配"
- 匹配度高的 patch 就是该查询负责的区域

### 5.6 上采样（Upscaling）

ViT 输出的特征图分辨率是原始图片的 1/16（512 -> 32）。这个分辨率做分类够了，但做精细的 mask 预测不够。

EoMT 会把特征图上采样到 1/4 分辨率（512 -> 128），然后再做 mask 预测。

```python
# 简单上采样
upscaled_features = F.interpolate(
    patch_features, 
    scale_factor=4,  # 从 1/16 到 1/4
    mode='bilinear', 
    align_corners=True
)
```

更高的分辨率意味着更精细的 mask 边界。

### 5.7 Masked Attention：训练时的约束

这是 EoMT 的另一个关键设计。

**问题：** 如果不加约束，100 个查询可能会"抢"同一个区域，或者互相干扰。

**解决方案：** 在训练时，用预测的 mask 来约束 Self-Attention。

具体做法：

1. 先用当前查询预测一个粗略的 mask
2. 把这个 mask 作为"注意力掩码"：每个查询只能关注它自己预测区域内的 patch
3. 查询和 patch 做 Self-Attention 时，只允许查询和它"自己的"patch 通信

**效果：** 每个查询被迫专注于自己的区域，不能"偷看"别人的地盘。

### 5.8 Mask Annealing：从严格到宽松

Masked Attention 虽然好，但有个问题：如果一开始预测的 mask 不准，约束反而有害。

**Mask Annealing 的思路：** 训练初期约束强一点，让查询学会"专注"。训练后期逐渐放松约束，让查询能"看到"全局信息，做更准确的判断。

```python
def get_mask_annealing_weight(epoch, num_epochs, power=0.9):
    # 多项式衰减：从 1.0 逐渐降到 0.0
    return (1 - epoch / num_epochs) ** power
```

- 训练初期（epoch=0）：weight=1.0，Masked Attention 完全生效
- 训练中期：weight 逐渐减小，约束逐渐放松
- 训练后期（epoch=num_epochs）：weight=0.0，Masked Attention 完全关闭

**直觉理解：**

先让每个查询"圈好自己的地盘"，学会在自己的地盘里做精细的特征提取。等它们都学会之后，再放开限制，让它们看看全局信息，做更准确的分类。

### 5.9 训练 vs 推理

| 阶段 | Masked Attention | 行为 |
|------|-----------------|------|
| 训练 | 有（带 annealing） | 查询先圈地，再逐渐放开 |
| 推理 | 无 | 查询自由关注所有区域 |

推理时不需要 Masked Attention，因为查询已经训练好了，知道该关注哪里。

### 5.10 Loss 函数设计

EoMT 的 Loss 由两部分组成：

**分类 Loss：**

每个查询要预测它负责区域的类别。用 CrossEntropyLoss，和普通分类一样。

```python
class_loss = F.cross_entropy(class_pred, target_classes)
```

但这里有个问题：我们不知道每个查询对应哪个类别。所以需要先做匹配（用匈牙利算法），把查询和真实物体一一配对，然后算 Loss。

**Mask Loss：**

每个查询预测的 mask 要和真实 mask 尽可能接近。用 Dice Loss 或 Binary Cross Entropy：

```python
def dice_loss(pred, target):
    smooth = 1.0
    pred = pred.contiguous().view(-1)
    target = target.contiguous().view(-1)
    intersection = (pred * target).sum()
    return 1 - (2. * intersection + smooth) / (pred.sum() + target.sum() + smooth)
```

Dice Loss 对类别不平衡不敏感。如果某个物体很小（比如远处的鸟），CrossEntropy 可能直接忽略它，但 Dice Loss 会"注意到"。

**总 Loss：**

```python
total_loss = class_loss + mask_loss
```

两个 Loss 的权重可以调整，但简单相加通常效果就不错。

### 5.11 为什么叫"Encoder-only"

传统 Transformer 做分割通常用 Encoder-Decoder 结构：

- Encoder 提取特征
- Decoder 把特征"翻译"成分割结果

EoMT 的作者发现：**如果 Encoder 足够强，Decoder 是多余的。** 直接在 Encoder 中间插入可学习查询，让 Encoder 自己完成"特征提取 + 分割"两件事。

这就像：别人需要两个专家（编码专家 + 解码专家）合作完成一个任务，EoMT 只用一个超级专家（编码器），给他几个"助手"（可学习查询），他自己就能搞定。

**好处：**

1. 参数更少（没有解码器）
2. 训练更快（少了一个大模块）
3. 充分利用预训练权重（DINOv3 的权重全部保留）

### 5.12 查询数量和图片内容的关系

100 个查询够用吗？这取决于图片的复杂度。

- 简单图片（只有 2-3 个物体）：100 个查询绰绰有余
- 复杂图片（街景，有几十个物体）：100 个查询可能不够

查询太少会漏掉小物体，查询太多会浪费计算资源。PASCAL VOC 每张图片平均有 2-3 个物体，100 个查询完全够用。

如果换到更复杂的数据集（比如 COCO，每张图可能有 10+ 个物体），可以适当增加查询数量。

### 5.13 查询初始化的影响

查询的初始值会影响训练结果吗？会的。

如果所有查询初始化为相同的值，它们一开始会"看到"相同的东西，需要很长时间才能"分化"成不同的探针。

实践中，nn.Embedding 默认用随机初始化，每个查询的初始向量都不同，这已经足够了。更高级的做法是用 K-means 聚类初始化，让查询一开始就分散在特征空间的不同位置。

### 5.14 为什么 EoMT 在 VOC 上效果好

PASCAL VOC 2012 有 20 个类别，每张图片通常只有 2-5 个物体。这个特点让 EoMT 特别适合：

1. **类别少**：20 个类别 + 1 个背景，分类任务相对简单
2. **物体少**：每张图只有几个物体，100 个查询绰绰有余
3. **物体大**：VOC 的物体通常比较大（不像 COCO 有很多小物体），1/16 分辨率也能覆盖

如果换到 COCO 数据集（80 个类别，每张图有很多小物体），EoMT 可能需要更多查询和更高的特征分辨率。

### 5.15 和 MaskFormer / Mask2Former 的关系

EoMT 属于"Mask Transformer"家族。这个家族的共同特点是：

1. 用一组可学习查询来表示不同的分割区域
2. 每个查询输出一个类别和一个 mask
3. 用匈牙利算法做匹配

**MaskFormer（2021）：** 最早提出这种范式，用 Transformer 解码器生成 mask

**Mask2Former（2022）：** 改进版，用 masked attention 让解码器更高效

**EoMT（2023）：** 去掉解码器，直接在编码器里做，更简单更高效

EoMT 可以看作是 MaskFormer 系列的"极简版"，它保留了核心思想（可学习查询 + mask 预测），但去掉了复杂的解码器设计。

### 6.1 阅读 model.py

EoMT 的模型文件主要包含：

```python
class EoMT(nn.Module):
    def __init__(self, num_classes=21, num_queries=100, num_blocks=6, ...):
        # 1. 加载 DINOv3 骨干网络
        # 2. 创建可学习查询 (nn.Embedding)
        # 3. 创建后面的 Transformer Block
        # 4. 创建 Class Head 和 Mask Head
    
    def forward(self, x):
        # 1. DINOv3 前 L1 层提取特征
        # 2. 插入查询 token
        # 3. 后 L2 层做 Self-Attention（训练时带 Masked Attention）
        # 4. 上采样特征
        # 5. Class Head 和 Mask Head 输出
        # 6. 点积生成 mask logits
```

**逐行理解 forward：**

1. 输入图片 `(B, 3, 512, 512)` 进入 DINOv3
2. DINOv3 的前 L1 层处理，输出中间特征 `(B, N, 384)`
3. 创建/插入查询 token `(B, num_q, 384)`，和 patch token 拼接
4. 拼接后的序列进入后 L2 层 Transformer Block
5. 从输出中分离查询特征和 patch 特征
6. patch 特征上采样到 1/4 分辨率
7. Class Head 输出类别 `(B, num_q, 21)`
8. Mask Head 输出 mask 嵌入 `(B, num_q, 384)`
9. 点积生成 mask logits `(B, num_q, H_upsampled * W_upsampled)`

### 6.6 如何调参

| 参数 | 作用 | 调大 | 调小 |

| `batch_size` | 每次喂多少张图 | 训练更稳，但更吃显存 | 省显存，但梯度噪声大 |

### 6.7 常见错误

**OOM（Out of Memory，显存不足）：**

```
RuntimeError: CUDA out of memory.
```

解决办法：减小 `batch_size`（4 -> 2 -> 1），或者减小图片尺寸。

**CUDA 错误：**

```
RuntimeError: CUDA error: device-side assert triggered
```

通常是类别数不匹配。检查 `num_classes` 是不是 21，检查标签值是不是在 0-20 范围内。

**Import 路径错误：**

```
ModuleNotFoundError: No module named 'dataset'
```

确保在项目根目录运行，或者添加路径：

```python
import sys, os
sys.path.insert(0, os.path.join(os.path.dirname(__file__), '..'))
```

### 6.8 调试技巧

训练过程中遇到问题怎么办？这里有一些实用的调试方法：

**Loss 不下降：**

检查学习率是不是太大或太小。学习率太大，Loss 会震荡不收敛；学习率太小，Loss 下降很慢。可以试试 1e-3、1e-4、1e-5 这几个值。

**Loss 变成 NaN：**

通常是梯度爆炸了。试试梯度裁剪：

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

**验证集 mIoU 很低但训练 Loss 正常：**

可能是过拟合了。试试增大 Dropout，或者加数据增强（随机翻转、颜色抖动等）。

**训练速度很慢：**

检查 DataLoader 的 `num_workers` 参数。设为 4 或 8 可以让数据加载和模型训练并行。

```python
loader = DataLoader(dataset, batch_size=4, shuffle=True, num_workers=4)
```
