### 1.1 实验目标

这个实验的目标很明确：**用 DINOv3 (ViT-S/384) 做骨干网络，加上 EoMT 分割头，在 PASCAL VOC 2012 数据集上做语义分割。**

1. 加载一个预训练好的 DINOv3 模型（它已经会"看"图片了）
2. 在它上面加一个 EoMT 分割头（让它学会"分割"图片）

`dataset.py` 里定义了一个 `VOCSegDataset` 类，继承自 PyTorch 官方的 `torchvision.datasets.VOCSegmentation`。它负责：

### 1.4 和 PSPNet 的关键区别

| 对比项 | PSPNet | EoMT |
|--------|--------|------|
| 分割头 | 金字塔池化 + 卷积层 | 可学习 Queries + Transformer Block |
| 训练策略 | 标准训练 | Mask Annealing（掩码退火） |
| 数据划分 | trainval/trainval（有误） | train/val（正确） |
| 输出形式 | 直接逐像素分类 | Queries 预测 Mask + 类别，再合并 |
| 推理速度 | 较慢 | 更快（纯 ViT 架构） |

- **一个训练好的模型**：`eomt_voc_weights.pth`
- **mIoU 评估指标**：在验证集上的语义分割精度
- **预测可视化**：输入一张图片，输出分割结果图

### 3.4 数据预处理详解

`dataset.py` 在 `__getitem__` 中做了三件事：

**第一步：缩放到 512x512**

```python
# 图片用双线性插值（平滑过渡，保留细节）
img = F.resize(img, (512, 512), interpolation=F.InterpolationMode.BILINEAR)

# 标签用最近邻插值（绝对不能平滑！）
mask = F.resize(mask, (512, 512), interpolation=F.InterpolationMode.NEAREST)
```

这里有个关键点：**标签图（mask）必须用 NEAREST 插值**。因为标签图里存的是类别编号（0, 1, 2, ...），如果用双线性插值，0 和 1 之间会插出 0.5，这就乱套了。NEAREST 插值保证每个像素的类别编号还是整数。

**第二步：转 Tensor 并标准化**

```python
img = F.to_tensor(img)  # (H, W, C) -> (C, H, W)，像素值归一化到 [0, 1]
img = F.normalize(img, mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
```

标准化用的是 ImageNet 的均值和标准差。DINOv3 是在 ImageNet 上预训练的，所以输入也要用同样的标准化参数。

**第三步：标签转 LongTensor**

```python
mask = torch.as_tensor(np.array(mask), dtype=torch.long)
```

CrossEntropyLoss 要求标签是 `torch.long` 类型。

## 第四章：EoMT 模型详解

### 4.1 架构回顾

EoMT 的架构可以用一句话说清楚：**一个 ViT + 一组可学习 Query + 一个极小的预测头**。

完整的流程：

```
输入图片 (3, 512, 512)
    ↓
Patch Embedding → (1024, 384)  ← 把图片切成 16x16 的 patch
    ↓
L1 Blocks × 8  ← 前 8 层 Transformer Block，纯特征提取
    ↓
拼接 Queries (100, 384)  ← 插入可学习查询向量
    ↓
L2 Blocks × 4  ← 后 4 层，Query 和 Patch 一起做 Self-Attention
    ↓
Class Head → 每个 Query 预测类别 (100, 22)
Mask Head → 每个 Query 预测 Mask (100, 128, 128)
```

核心创新在于：**把 Query 直接拼进 Encoder，让 Self-Attention 同时做三件事**：

1. Query 之间互相通信（协调各自负责的区域）
2. Query 从 Patch 读取信息（提取区域特征）
3. Patch 之间继续做特征提取（保持图像理解）

这三种交互在同一个 MHSA 操作中并行完成，非常高效。

详细的架构讲解见《01_论文精讲_EoMT.md》第五章。

### 4.2 完整 model.py 代码

以下是 `model.py` 的完整代码，可以直接复制运行：

```python
import math
import torch
import torch.nn as nn
import torch.nn.functional as F
import timm


class LayerNorm2d(nn.LayerNorm):
    """LayerNorm for 2D feature maps (channel-only normalization)."""

    def __init__(self, num_channels, eps=1e-6):
        super().__init__(num_channels, eps=eps)

    def forward(self, x):
        # x: (B, C, H, W) -> permute to (B, H, W, C) -> LayerNorm -> permute back
        x = x.permute(0, 2, 3, 1)
        x = super().forward(x)
        x = x.permute(0, 3, 1, 2)
        return x


class ScaleBlock(nn.Module):
    """Upscales feature map 2x using transposed conv + depthwise conv + LayerNorm."""

    def __init__(self, embed_dim):
        super().__init__()
        self.conv1 = nn.ConvTranspose2d(embed_dim, embed_dim, kernel_size=2, stride=2)
        self.act = nn.GELU()
        self.conv2 = nn.Conv2d(
            embed_dim, embed_dim, kernel_size=3, padding=1,
            groups=embed_dim, bias=False,
        )
        self.norm = LayerNorm2d(embed_dim)

    def forward(self, x):
        x = self.conv1(x)
        x = self.act(x)
        x = self.conv2(x)
        x = self.norm(x)
        return x


class EoMT(nn.Module):
    """Encoder-only Mask Transformer for semantic segmentation."""

    def __init__(self, num_classes=21, num_q=100, num_blocks=4,
                 pretrained=True, masked_attn_enabled=True):
        super().__init__()
        self.num_classes = num_classes
        self.num_q = num_q
        self.num_blocks = num_blocks
        self.masked_attn_enabled = masked_attn_enabled

        # -- Backbone: DINOv3 ViT-S/16 ------------------------------------
        self.backbone = timm.create_model(
            'vit_small_patch16_dinov3.lvd1689m',
            pretrained=pretrained,
            num_classes=0,
        )
        # Freeze backbone
        for p in self.backbone.parameters():
            p.requires_grad = False

        embed_dim = self.backbone.embed_dim  # 384
        patch_size = self.backbone.patch_embed.patch_size  # (16, 16)
        self.patch_size = patch_size

        # Number of prefix tokens (cls + registers)
        self.num_prefix_tokens = getattr(
            self.backbone, 'num_prefix_tokens',
            1 + 4,  # fallback: 1 cls + 4 register tokens for DINOv3
        )

        # -- Learnable queries --------------------------------------------
        self.q = nn.Embedding(num_q, embed_dim)

        # -- Prediction heads ---------------------------------------------
        self.class_head = nn.Linear(embed_dim, num_classes + 1)  # +1 for "no object"

        self.mask_head = nn.Sequential(
            nn.Linear(embed_dim, embed_dim), nn.GELU(),
            nn.Linear(embed_dim, embed_dim), nn.GELU(),
            nn.Linear(embed_dim, embed_dim),
        )

        # -- Upscale blocks -----------------------------------------------
        num_upscale = max(1, int(math.log2(max(patch_size))) - 2)
        # patch_size=16 -> max(1, 4-2) = 2 ScaleBlocks (16->32->64)
        self.upscale = nn.Sequential(
            *[ScaleBlock(embed_dim) for _ in range(num_upscale)]
        )

        # -- Annealing buffer ---------------------------------------------
        self.register_buffer("attn_mask_probs", torch.ones(num_blocks))

    # -- Forward helpers --------------------------------------------------

    def _get_grid_size(self, num_patch_tokens):
        """Compute spatial grid size from number of patch tokens."""
        h = w = int(math.isqrt(num_patch_tokens))
        if h * w != num_patch_tokens:
            return self.backbone.patch_embed.grid_size
        return (h, w)

    def _predict(self, x):
        """Predict masks and class logits from current features."""
        q = x[:, :self.num_q, :]
        class_logits = self.class_head(q)

        # Extract patch tokens (skip queries + prefix tokens)
        x = x[:, self.num_q + self.num_prefix_tokens:, :]
        # Reshape to spatial: (B, C, H, W)
        grid_size = self._get_grid_size(x.shape[1])
        x = x.transpose(1, 2).reshape(x.shape[0], -1, *grid_size)

        # Mask logits via dot product
        mask_logits = torch.einsum(
            "bqc,bchw->bqhw",
            self.mask_head(q),
            self.upscale(x),
        )

        return mask_logits, class_logits

    def _build_attn_mask(self, x, mask_logits):
        """Build attention mask from mask logits (query->patch only)."""
        B, N, _ = x.shape
        attn_mask = torch.ones(B, N, N, dtype=torch.bool, device=x.device)

        num_patches = N - self.num_q - self.num_prefix_tokens
        grid_size = self._get_grid_size(num_patches)
        interpolated = F.interpolate(mask_logits, grid_size, mode='bilinear')
        interpolated = interpolated.view(B, self.num_q, -1)

        # Only mask query->patch attention
        patch_start = self.num_q + self.num_prefix_tokens
        attn_mask[:, :self.num_q, patch_start:] = (interpolated > 0)

        return attn_mask

    def _disable_attn_mask(self, attn_mask, prob):
        """Randomly disable masking for some queries (annealing)."""
        if prob < 1:
            random_queries = torch.rand(
                attn_mask.shape[0], self.num_q, device=attn_mask.device
            ) > prob
            patch_start = self.num_q + self.num_prefix_tokens
            attn_mask[:, :self.num_q, patch_start:][random_queries] = True
        return attn_mask

    def _custom_attn(self, attn_module, x, mask):
        """Custom attention forward handling timm version differences."""
        B, N, C = x.shape
        qkv = attn_module.qkv(x).reshape(
            B, N, 3, attn_module.num_heads, -1
        )
        q, k, v = qkv.permute(2, 0, 3, 1, 4).unbind(0)

        if mask is not None:
            mask = mask[:, None, :, :]  # (B, 1, N, N) for broadcast over heads

        x = F.scaled_dot_product_attention(q, k, v, attn_mask=mask)
        x = attn_module.proj(x.transpose(1, 2).reshape(B, N, C))
        return x

    # -- Main forward -----------------------------------------------------

    def forward(self, x):
        # NOTE: Input normalization is handled by dataset.py
        # Alternative: uncomment below if dataset.py does NOT normalize
        # x = (x - self.pixel_mean) / self.pixel_std

        x = self.backbone.patch_embed(x)

        # Position embedding (handle timm version differences)
        if hasattr(self.backbone, '_pos_embed'):
            x, _ = self.backbone._pos_embed(x)
        elif hasattr(self.backbone, 'pos_embed'):
            x = x + self.backbone.pos_embed

        attn_mask = None
        mask_logits_list, class_logits_list = [], []

        for i, block in enumerate(self.backbone.blocks):
            # Insert queries at L1/L2 boundary
            if i == len(self.backbone.blocks) - self.num_blocks:
                x = torch.cat([
                    self.q.weight[None, :, :].expand(x.shape[0], -1, -1),
                    x,
                ], dim=1)

            # Predict intermediate masks for masked attention (training only)
            if self.masked_attn_enabled and \
               i >= len(self.backbone.blocks) - self.num_blocks:
                mask_logits, class_logits = self._predict(
                    self.backbone.norm(x)
                )
                mask_logits_list.append(mask_logits)
                class_logits_list.append(class_logits)

                attn_mask = self._build_attn_mask(x, mask_logits)
                block_idx = i - (len(self.backbone.blocks) - self.num_blocks)
                attn_mask = self._disable_attn_mask(
                    attn_mask, self.attn_mask_probs[block_idx]
                )

            # Forward through block
            attn = block.attn if hasattr(block, 'attn') else block.attention
            attn_out = self._custom_attn(attn, block.norm1(x), attn_mask)

            # Handle layer scaling (timm version differences)
            if hasattr(block, 'ls1'):
                x = x + block.ls1(attn_out)
            elif hasattr(block, 'layer_scale1'):
                x = x + block.layer_scale1(attn_out)
            elif hasattr(block, 'gamma_1'):
                x = x + block.gamma_1 * attn_out
            else:
                x = x + attn_out

            mlp_out = block.mlp(block.norm2(x))
            if hasattr(block, 'ls2'):
                x = x + block.ls2(mlp_out)
            elif hasattr(block, 'layer_scale2'):
                x = x + block.layer_scale2(mlp_out)
            elif hasattr(block, 'gamma_2'):
                x = x + block.gamma_2 * mlp_out
            else:
                x = x + mlp_out

        # Final prediction
        mask_logits, class_logits = self._predict(self.backbone.norm(x))
        mask_logits_list.append(mask_logits)
        class_logits_list.append(class_logits)

        return mask_logits_list, class_logits_list


if __name__ == '__main__':
    model = EoMT(num_classes=21)
    model.eval()
    x = torch.randn(1, 3, 512, 512)
    mask_logits, class_logits = model(x)
    print(f"EoMT forward pass OK")
    print(f"  Mask logits: {len(mask_logits)} layers, final shape {mask_logits[-1].shape}")
    print(f"  Class logits: {len(class_logits)} layers, final shape {class_logits[-1].shape}")
```

### 4.3 ScaleBlock 详解

`ScaleBlock` 的作用是把特征图的分辨率翻倍。ViT 输出的特征图是 1/16 分辨率（32x32），我们需要把它上采样到 1/4 分辨率（128x128），这样才能做精细的 Mask 预测。

为什么用转置卷积而不是简单的 `F.interpolate`？因为转置卷积是**可学习的**，它可以根据数据自动学习如何上采样，效果比固定插值好。具体来说，`nn.ConvTranspose2d` 会在每个像素之间插入空白，然后用可学习的卷积核填充这些空白，相当于在学习"如何把低分辨率图像变清晰"。

为什么用 depthwise conv？因为 depthwise conv 对每个通道独立做卷积，参数量只有普通卷积的 1/通道数，非常轻量。它在这里的作用是做空间平滑，消除转置卷积可能产生的棋盘格效应。棋盘格效应是转置卷积的常见问题，表现为输出图像上有规律的格子状伪影。depthwise conv 的 3x3 卷积核可以平滑这些伪影。

为什么用 LayerNorm2d？因为 Transformer 习惯用 LayerNorm 而不是 BatchNorm。LayerNorm 对每个位置独立做归一化，不依赖 batch 大小，对小 batch 更友好。BatchNorm 在 batch_size=1 时会有严重问题（无法计算均值和方差），而 LayerNorm 没有这个问题。我们的训练 batch_size 只有 2，用 LayerNorm 更稳妥。

整个 ScaleBlock 的流程可以这样理解：先用转置卷积把分辨率翻倍（同时可能引入棋盘格噪声），然后用 GELU 激活增加非线性，再用 depthwise conv 做平滑去噪，最后用 LayerNorm 做归一化稳定训练。每一步都有明确的设计意图。

### 4.4 Query 机制详解

Query 的实现极其简单：

```python
self.q = nn.Embedding(num_q, embed_dim)
```

就是一个可学习的 Embedding 矩阵，形状为 `(100, 384)`。训练开始时随机初始化，训练过程中不断调整。

**为什么是 100 个 Query？**

PASCAL VOC 有 21 个类别（20 类物体 + 1 背景），每张图片通常只有 2-5 个物体。100 个 Query 远远够用。多出来的 Query 会学会关注"无对象"区域（通过 `num_classes + 1` 中的 "+1" 类别来处理）。

你可能想问：为什么不用 21 个 Query，每个类别一个？因为一张图片里可能有多个同类物体（比如两只猫），每个物体需要独立的 Query 来覆盖。而且 Query 和类别不是一一对应的关系，一个 Query 可能学会覆盖多个同类区域，也可能多个 Query 协作覆盖一个大物体。100 个 Query 给了模型足够的灵活性。

**Query 插入的位置：**

在 ViT 的第 8 个 Block 之后（L1=8, L2=4），Query 被拼接到 Token 序列的末尾：

```
[queries(100), cls_token(1), register_tokens(4), patch_tokens(1024)]
```

拼接后的序列长度为 100 + 1 + 4 + 1024 = 1129。

**Query 在 Self-Attention 中的行为：**

插入 Query 后，Self-Attention 的计算方式不变，但参与计算的 Token 类型变多了。具体来说：

- Query 和 Query 之间做 Self-Attention：相当于让 100 个"探针"互相通信，协调各自负责的区域。如果两个 Query 不小心关注了同一个区域，它们可以通过这种通信互相"推开"。
- Query 和 Patch 之间做 Attention：Query 从 Patch Token 中读取图像特征信息。每个 Query 会学会只关注和自己区域相关的 Patch。
- Patch 和 Patch 之间做 Self-Attention：和之前一样，继续做正常的图像特征提取。

这三种交互在同一个 MHSA 操作中并行完成，不需要额外的 Cross-Attention 模块。这是 EoMT 高效的核心原因。

### 4.5 两种损失函数详解

本项目实现了两种损失函数，默认使用简化版。

#### 简化版（默认）：逐像素 CE + Dice Loss

这是本项目默认使用的损失函数。核心思路是把 EoMT 的 `(B, num_q, H, W)` 输出转换为 `(B, C, H, W)` 的逐像素预测，然后和标准语义分割一样算损失。

```python
# 核心转换代码
mask_probs = mask_logits.sigmoid()           # (B, 100, 128, 128)
class_probs = class_logits.softmax(dim=-1)   # (B, 100, 22)
# 去掉 "no object" 类别，合并成逐像素预测
per_pixel = torch.einsum("bqhw,bqc->bchw", mask_probs, class_probs[..., :-1])
# 上采样到 512x512 和标签对齐
per_pixel = F.interpolate(per_pixel, size=(512, 512), mode='bilinear')

# CE Loss
ce_loss = CrossEntropyLoss(ignore_index=255)(per_pixel, masks)

# Dice Loss（对每个类别算 Dice，然后平均）
dice_loss = 0.0
for c in range(21):
    pred_c = (pred == c).float()
    mask_c = (masks == c).float()
    intersection = (pred_c * mask_c).sum()
    dice_loss += 1 - (2 * intersection + 1) / (pred_c.sum() + mask_c.sum() + 1)
dice_loss = dice_loss / 21

total_loss = ce_loss + dice_loss
```

`torch.einsum("bqhw,bqc->bchw", ...)` 这行是关键。它的意思是：对每个 Query q，用它的 Mask 概率 `mask_probs[b, q, h, w]` 作为权重，把它的类别概率 `class_probs[b, q, c]` 分配到对应的像素上。这样就把 100 个 Query 的预测合并成了 21 个类别的逐像素预测。

#### 完整版（进阶）：匈牙利匹配 + BCE + Dice + CE

完整版更接近论文原始实现。核心区别在于：**先做匹配，再算损失**。

匹配过程：

1. 对每个 Query，计算它和每个真实物体的"代价"（cost）
2. 代价由三部分组成：分类代价（CE）、Mask 代价（BCE + Dice）、位置代价
3. 用匈牙利算法找到最优的 Query-物体配对
4. 只对匹配上的 Query 算损失，没匹配上的 Query 算"无对象"损失

```python
# 伪代码：匈牙利匹配
from scipy.optimize import linear_sum_assignment

# 计算代价矩阵 (num_q, num_objects)
cost_matrix = torch.zeros(num_q, num_objects)
for i in range(num_q):
    for j in range(num_objects):
        class_cost = F.cross_entropy(class_logits[i], gt_class[j])
        mask_cost = dice_loss(mask_pred[i], gt_mask[j]) + \
                    F.binary_cross_entropy(mask_pred[i], gt_mask[j])
        cost_matrix[i, j] = class_cost + mask_cost

# 匈牙利算法求解最优匹配
q_indices, obj_indices = linear_sum_assignment(cost_matrix.cpu().numpy())
```

完整版的参考代码在：`github.com/tue-mps/eomt/training/mask_classification_loss.py`

## 第五章：训练指南

### 5.1 完整 train.py 代码

```python
import sys
import os
sys.path.insert(0, os.path.join(os.path.dirname(__file__), '..'))

import torch
import torch.nn as nn
import torch.nn.functional as F
import torch.optim as optim
from torch.utils.data import DataLoader
import numpy as np

from dataset import VOCSegDataset
from eomt.model import EoMT


class EoMTLoss(nn.Module):
    """Simplified per-pixel loss (no Hungarian matching)."""

    def __init__(self, num_classes=21, ce_weight=1.0, dice_weight=1.0):
        super().__init__()
        self.ce_weight = ce_weight
        self.dice_weight = dice_weight
        self.ce = nn.CrossEntropyLoss(ignore_index=255)

    def forward(self, mask_logits, class_logits, masks):
        """
        mask_logits: (B, num_q, H, W) - raw logits
        class_logits: (B, num_q, num_classes+1) - raw logits
        masks: (B, H, W) - ground truth labels (0-20, 255=ignore)
        """
        # Convert to per-pixel predictions
        mask_probs = mask_logits.sigmoid()  # (B, num_q, H, W)
        class_probs = class_logits.softmax(dim=-1)  # (B, num_q, C+1)
        # Remove "no object" class for per-pixel
        per_pixel = torch.einsum("bqhw,bqc->bchw", mask_probs, class_probs[..., :-1])

        # Upsample to match ground truth resolution (128->512 for VOC)
        per_pixel = F.interpolate(
            per_pixel, size=masks.shape[-2:], mode='bilinear', align_corners=False
        )

        # Cross-entropy loss
        ce_loss = self.ce(per_pixel, masks)

        # Dice loss (simplified, per-class)
        pred = torch.argmax(per_pixel, dim=1)  # (B, H, W)
        smooth = 1.0
        dice_loss = 0.0
        for c in range(per_pixel.shape[1]):  # num_classes
            pred_c = (pred == c).float()
            mask_c = (masks == c).float()
            intersection = (pred_c * mask_c).sum()
            dice_loss += 1 - (2 * intersection + smooth) / \
                         (pred_c.sum() + mask_c.sum() + smooth)
        dice_loss = dice_loss / per_pixel.shape[1]

        return self.ce_weight * ce_loss + self.dice_weight * dice_loss


def mask_annealing(start_step, current_step, end_step, power=3.0):
    """Polynomial annealing from 1.0 to 0.0."""
    if current_step < start_step:
        return 1.0
    elif current_step >= end_step:
        return 0.0
    progress = (current_step - start_step) / (end_step - start_step)
    return (1.0 - progress) ** power


def fast_hist(a, b, n):
    k = (a >= 0) & (a < n)
    return np.bincount(n * a[k].astype(int) + b[k], minlength=n ** 2).reshape(n, n)


def validate(model, val_loader, device):
    model.eval()
    hist = np.zeros((21, 21))
    with torch.no_grad():
        for images, masks in val_loader:
            images = images.to(device)
            mask_logits_list, class_logits_list = model(images)
            # Convert to per-pixel
            mask_probs = mask_logits_list[-1].sigmoid()
            class_probs = class_logits_list[-1].softmax(dim=-1)
            per_pixel = torch.einsum(
                "bqhw,bqc->bchw", mask_probs, class_probs[..., :-1]
            )
            per_pixel = F.interpolate(
                per_pixel, size=masks.shape[-2:], mode='bilinear', align_corners=False
            )
            preds = torch.argmax(per_pixel, dim=1)
            for b in range(preds.shape[0]):
                hist += fast_hist(
                    masks[b].cpu().numpy().flatten(),
                    preds[b].cpu().numpy().flatten(),
                    21,
                )
    iou = np.diag(hist) / (hist.sum(1) + hist.sum(0) - np.diag(hist) + 1e-10)
    miou = np.nanmean(iou)
    return miou


def train(batch_size=2, num_epochs=50, learning_rate=1e-3, num_q=100,
          num_blocks=4, pretrained=True, dry_run=False):
    # -- Device -----------------------------------------------------------
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    print(f"Device: {device}")

    # -- Dataset ----------------------------------------------------------
    print("Loading PASCAL VOC 2012 dataset...")
    train_dataset = VOCSegDataset(
        root='../voc_data', year='2012', image_set='train', download=False,
    )
    val_dataset = VOCSegDataset(
        root='../voc_data', year='2012', image_set='val', download=False,
    )
    train_loader = DataLoader(
        train_dataset, batch_size=batch_size, shuffle=True, drop_last=True,
    )
    val_loader = DataLoader(
        val_dataset, batch_size=batch_size, shuffle=False, drop_last=False,
    )
    print(f"Train: {len(train_dataset)} images, {len(train_loader)} batches")
    print(f"Val:   {len(val_dataset)} images, {len(val_loader)} batches")

    # -- Model ------------------------------------------------------------
    model = EoMT(
        num_classes=21, num_q=num_q, num_blocks=num_blocks,
        pretrained=pretrained, masked_attn_enabled=True,
    ).to(device)
    model.train()

    # -- Optimizer & Loss -------------------------------------------------
    optimizer = optim.AdamW(
        filter(lambda p: p.requires_grad, model.parameters()), lr=learning_rate,
    )
    criterion = EoMTLoss(num_classes=21)

    # -- Annealing schedule -----------------------------------------------
    # VOC train ~1464 images, batch_size=2 -> ~732 steps/epoch
    annealing_start = [0, 5856, 10248, 14640]    # block 0-3
    annealing_end = [8784, 14640, 20496, 26352]  # block 0-3

    # -- Training loop ----------------------------------------------------
    global_step = 0
    best_miou = 0.0
    for epoch in range(num_epochs):
        model.train()
        epoch_loss = 0.0

        for images, masks in train_loader:
            images, masks = images.to(device), masks.to(device)

            optimizer.zero_grad()
            mask_logits_list, class_logits_list = model(images)

            # Use final layer output
            loss = criterion(mask_logits_list[-1], class_logits_list[-1], masks)
            loss.backward()
            optimizer.step()

            # Update mask annealing probabilities
            for i in range(model.num_blocks):
                model.attn_mask_probs[i] = mask_annealing(
                    annealing_start[i], global_step, annealing_end[i],
                )

            epoch_loss += loss.item()
            global_step += 1

        avg_loss = epoch_loss / len(train_loader)
        print(f"Epoch [{epoch + 1}/{num_epochs}] Loss: {avg_loss:.4f}  "
              f"Step: {global_step}  "
              f"Mask probs: {model.attn_mask_probs.tolist()}")

        # Validation every 5 epochs
        if (epoch + 1) % 5 == 0:
            miou = validate(model, val_loader, device)
            print(f"  >>> Val mIoU: {miou:.4f} (best: {max(best_miou, miou):.4f})")
            if miou > best_miou:
                best_miou = miou
                torch.save(model.state_dict(), "eomt_voc_weights.pth")
                print(f"  >>> Saved best weights (mIoU={miou:.4f})")

        # Dry run: break after 1 epoch
        if dry_run:
            print("[DRY RUN] Breaking after 1 epoch")
            break

    print(f"Training complete. Best val mIoU: {best_miou:.4f}")


if __name__ == '__main__':
    import argparse
    parser = argparse.ArgumentParser()
    parser.add_argument('--dry-run', action='store_true', help='Run 1 epoch then exit')
    args = parser.parse_args()
    train(dry_run=args.dry_run)
```

### 5.4 Mask Annealing 详解

Mask Annealing 是 EoMT 训练的核心策略。它的作用是：**训练初期用 Masked Attention 帮助收敛，训练后期逐渐去掉 Mask，让模型学会在没有 Mask 的情况下也能正常工作**。

**为什么需要 Masked Attention？**

训练刚开始时，Query 是随机初始化的，它们完全不知道应该关注图像的哪个区域。如果不加约束，100 个 Query 可能会全部关注同一个区域（比如图片中央），或者互相干扰导致训练不稳定。Masked Attention 解决了这个问题：每个 Query 先预测一个粗糙的中间 Mask，然后在 Self-Attention 中，这个 Query 只能关注自己 Mask 区域内的 Patch Token。这相当于给每个 Query 划分了"势力范围"，让它们各司其职。

**为什么推理时要去掉 Masked Attention？**

推理时保留 Masked Attention 有两个问题。第一，每个 Block 都要额外算一次 Mask Head 和 Upscale，这很耗时。第二，Masked Attention 需要修改 Attention 的计算逻辑，没法直接用 FlashAttention 等高效实现。所以推理时去掉 Masked Attention 可以大幅提升速度。

**为什么用多项式退火？**

多项式退火比线性退火更平滑。在训练初期，退火速度慢，让 Query 有足够时间学会"圈地"。训练后期，退火速度快，让 Query 快速适应"无约束"状态。

公式：`P_mask = (1 - progress)^power`，其中 `power=3.0`。

`power=3.0` 意味着退火曲线是"先慢后快"的。训练进行到 50% 时，P_mask 还有 0.125（而不是线性退火的 0.5），这意味着 Mask 在训练的大部分时间里都保持较强的约束。这符合直觉：我们希望 Query 在充分学会"圈地"之后，再逐渐放开约束。

**为什么需要错开？**

4 个 L2 Block 各有不同的退火时间表：

| Block | 开始步数 | 结束步数 | 持续时间 |
|-------|---------|---------|---------|
| Block 0（最浅） | 0 | 8784 | 8784 步 |
| Block 1 | 5856 | 14640 | 8784 步 |
| Block 2 | 10248 | 20496 | 10248 步 |
| Block 3（最深） | 14640 | 26352 | 11712 步 |

错开的直觉：浅层 Block 的 Query 更早学会稳定的注意力模式，可以更早摆脱 Mask 约束。深层 Block 的 Query 还在精细调整，需要更长时间的 Mask 保护。

你可以这样理解：4 个 Block 就像 4 个学生，Block 0 是学霸，学得快可以早点"放养"；Block 3 是学渣，需要更长时间的"辅导"。错开退火让每个 Block 按照自己的节奏学习。

**退火参数：**

- `power=3.0`：多项式指数，控制退火曲线的形状
- 每个 Block 的 start/end steps：基于 batch_size=2 和 1464 张训练图片计算（约 732 步/epoch）

**退火过程的实际效果：**

训练开始时，所有 Block 的 `attn_mask_probs` 都是 1.0，Masked Attention 完全生效。随着训练进行，Block 0 的 prob 最先开始下降，然后是 Block 1、Block 2，最后是 Block 3。到训练结束时，所有 prob 都降到 0.0，Masked Attention 完全关闭。

在推理时，我们设置 `masked_attn_enabled=False`，模型完全不计算 Masked Attention，直接做纯 ViT 前向传播。这就是 EoMT 推理速度快的原因。

### 5.6 常见训练问题

**Loss 不下降**

可能原因：学习率太大（Loss 震荡）或太小（Loss 下降极慢）。尝试调整学习率到 1e-3、1e-4、1e-5 分别测试。

**OOM（显存不足）**

```
RuntimeError: CUDA out of memory.
```

减小 batch_size 到 1。如果还不行，检查是否有其他程序占用显存。

**NaN Loss**

通常是梯度爆炸。可以在 `loss.backward()` 之后加梯度裁剪：

```python
loss.backward()
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
optimizer.step()
```

**导入错误**

```
ModuleNotFoundError: No module named 'dataset'
```

确保在 `eomt/` 目录下运行。train.py 已经通过 `sys.path.insert(0, '..')` 添加了上级目录到 Python 路径。

**Proxy 错误**

如果遇到 `httpx` 或网络相关错误，可能是代理环境变量导致的：

```bash
unset HTTP_PROXY HTTPS_PROXY http_proxy https_proxy ALL_PROXY
```

### 6.1 完整 evaluate.py 代码

```python
import sys
import os
sys.path.insert(0, os.path.join(os.path.dirname(__file__), '..'))

import torch
import numpy as np
from torch.utils.data import DataLoader
from tqdm import tqdm

def fast_hist(a, b, n):
    """Build confusion matrix from flattened predictions and ground truth."""
    k = (a >= 0) & (a < n)
    return np.bincount(n * a[k].astype(int) + b[k], minlength=n ** 2).reshape(n, n)


def evaluate(weight_path="eomt_voc_weights.pth", batch_size=2):
    # -- Setup ------------------------------------------------------------
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    num_classes = 21

    model.load_state_dict(torch.load(weight_path, map_location=device))
    model.eval()

    # -- Evaluation loop --------------------------------------------------
    hist = np.zeros((num_classes, num_classes))

    print("考试开始！请耐心等待模型做完所有的题...")
    with torch.no_grad():
        for images, masks in tqdm(val_loader, desc="Evaluating"):
            images = images.to(device)
            masks = masks.to(device)

            mask_logits_list, class_logits_list = model(images)

            # Use final layer predictions
            mask_logits = mask_logits_list[-1]
            class_logits = class_logits_list[-1]

            # Convert EoMT output to per-pixel predictions
            mask_probs = mask_logits.sigmoid()          # (B, num_q, H, W)
            class_probs = class_logits.softmax(dim=-1)  # (B, num_q, 22)
            # Remove "no object" class for per-pixel
            per_pixel = torch.einsum("bqhw,bqc->bchw", mask_probs, class_probs[..., :-1])
            preds = torch.argmax(per_pixel, dim=1)      # (B, H, W)

            # Accumulate confusion matrix
            for b in range(preds.shape[0]):
                hist += fast_hist(
                    masks[b].cpu().numpy().flatten(),
                    preds[b].cpu().numpy().flatten(),
                    num_classes,
                )

    # -- Metrics ----------------------------------------------------------
    pixel_accuracy = np.diag(hist).sum() / hist.sum()

    iou_per_class = np.diag(hist) / (
        hist.sum(1) + hist.sum(0) - np.diag(hist) + 1e-10
    )

    valid = ~np.isnan(iou_per_class)
    miou = np.nanmean(iou_per_class[valid])

    # -- Output -----------------------------------------------------------
    class_names = [
        'Background', 'Aeroplane', 'Bicycle', 'Bird', 'Boat', 'Bottle',
        'Bus', 'Car', 'Cat', 'Chair', 'Cow', 'Diningtable', 'Dog', 'Horse',
        'Motorbike', 'Person', 'Pottedplant', 'Sheep', 'Sofa', 'Train', 'Tvmonitor'
    ]

    print("\n" + "=" * 40)
    print("EoMT 语义分割成绩单")
    print("=" * 40)
    print(f"全局像素准确率 (Pixel Accuracy): {pixel_accuracy * 100:.2f}%")
    print(f"平均交并比 (mIoU): {miou * 100:.2f}%")
    print("-" * 40)
    print("详细类别 IoU 得分：")
    for i in range(num_classes):
        if valid[i]:
            print(f" - {class_names[i].ljust(15)}: {iou_per_class[i] * 100:.2f}%")
    print("=" * 40)


if __name__ == '__main__':
    evaluate()
```

import os
import sys
import torch
import numpy as np
import matplotlib.pyplot as plt
from PIL import Image

# 添加项目根目录到 sys.path，以便导入 dataset.py

# ==========================================
# 1. PASCAL VOC 的调色板 (给不同物体涂上专属颜色)
# ==========================================
VOC_COLORMAP = [
    [0, 0, 0], [128, 0, 0], [0, 128, 0], [128, 128, 0],
    [0, 0, 128], [128, 0, 128], [0, 128, 128], [128, 128, 128],
    [64, 0, 0], [192, 0, 0], [64, 128, 0], [192, 128, 0],
    [64, 0, 128], [192, 0, 128], [64, 128, 128], [192, 128, 128],
    [0, 64, 0], [128, 64, 0], [0, 192, 0], [128, 192, 0],
    [0, 64, 128]
]


def decode_segmap(image, nc=21):
    r = np.zeros_like(image).astype(np.uint8)
    g = np.zeros_like(image).astype(np.uint8)
    b = np.zeros_like(image).astype(np.uint8)
    for l in range(0, nc):
        idx = image == l
        r[idx] = VOC_COLORMAP[l][0]
        g[idx] = VOC_COLORMAP[l][1]
        b[idx] = VOC_COLORMAP[l][2]
    return np.stack([r, g, b], axis=2)


def predict(image_path, weight_path="eomt_voc_weights.pth"):
    # -- 错误处理 ---------------------------------------------------------
    if not os.path.exists(weight_path):
        print(f"权重文件 {weight_path} 不存在！")
        print("请先运行 python train.py 训练模型。")
        return
    if not os.path.exists(image_path):
        print(f"图片文件 {image_path} 不存在！")
        return

    # -- 设备 -------------------------------------------------------------
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    print(f"使用设备: {device}")

    # -- 加载模型 ---------------------------------------------------------
    model = EoMT(num_classes=21, pretrained=False, masked_attn_enabled=False).to(device)
    model.load_state_dict(torch.load(weight_path, map_location=device))
    model.eval()

    # -- 图片预处理 -------------------------------------------------------
    raw_image = Image.open(image_path).convert('RGB')
    transform = transforms.Compose([
        transforms.Resize((512, 512)),
        transforms.ToTensor(),
        transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
    ])
    img_tensor = transform(raw_image).unsqueeze(0).to(device)

    # -- 推理 -------------------------------------------------------------
    print("推理中...")
    with torch.no_grad():
        mask_logits_list, class_logits_list = model(img_tensor)

    # 转换为逐像素预测
    mask_probs = mask_logits_list[-1].sigmoid()
    class_probs = class_logits_list[-1].softmax(dim=-1)
    per_pixel = torch.einsum("bqhw,bqc->bchw", mask_probs, class_probs[..., :-1])
    pred = torch.argmax(per_pixel, dim=1).squeeze().cpu().numpy()
    colorized_pred = decode_segmap(pred)

    # -- 可视化 -----------------------------------------------------------
    fig, axes = plt.subplots(1, 2, figsize=(12, 6))
    axes[0].imshow(raw_image.resize((512, 512)))
    axes[0].set_title('Original Image')
    axes[0].axis('off')

    axes[1].imshow(colorized_pred)
    axes[1].set_title('EoMT Prediction')
    axes[1].axis('off')

    plt.tight_layout()
    plt.savefig("result.png")
    print("预测完成！结果已保存为 result.png")


if __name__ == "__main__":
    predict("test.jpg")

## 第七章：进阶与扩展

### 7.1 微调 Backbone

默认情况下，DINOv3 backbone 是冻结的（`requires_grad = False`），只训练 EoMT head。这样做的原因是：DINOv3 已经在 14 亿张图片上预训练好了，它的特征已经非常通用，我们只需要在它上面加一个"小头"做分割即可。冻结 backbone 可以节省大量显存和计算时间。

但如果你想追求更高的精度，可以尝试微调 backbone。微调的意思是让 backbone 的参数也参与训练，让它针对语义分割任务做细微调整。

**在 model.py 中注释掉冻结代码：**

```python
# 找到 model.py 中这几行，注释掉
# for p in self.backbone.parameters():
#     p.requires_grad = False
```

**在 train.py 中降低学习率：**

```python
# 微调 backbone 时，学习率要小很多
train(learning_rate=1e-5, batch_size=1)
```

为什么学习率要小很多？因为 backbone 的预训练权重已经很好了，我们不希望大幅改动它。学习率太大可能会破坏预训练学到的特征。1e-5 是一个安全的起点。

**预期效果：**

- mIoU 提升 1-3%
- 显存需求增加（batch_size 可能需要降到 1）
- 训练时间变长（因为需要更新的参数变多了）

**注意事项：**

微调 backbone 时，建议先用冻结模式训练 30 个 epoch，等 EoMT head 基本收敛后，再解冻 backbone 用更小的学习率微调 10-20 个 epoch。这种"先冻后调"的策略比从头微调效果更好。

### 7.2 匈牙利匹配完整版

从简化版迁移到完整版的主要改动在 `train.py` 中：

1. 替换 `EoMTLoss` 为匈牙利匹配版本
2. 在损失函数中加入 `scipy.optimize.linear_sum_assignment` 做匹配
3. 匹配后分别计算分类损失（CE）和 Mask 损失（BCE + Dice）

**为什么需要匈牙利匹配？**

简化版把 EoMT 的输出合并成逐像素预测，然后算 CE + Dice。这种方法简单直接，但有个问题：它没有显式地让每个 Query 学会关注特定的物体。匈牙利匹配解决了这个问题：它先把 100 个 Query 和图片中的真实物体一一配对，然后让每个 Query 只对自己配对的物体负责。

**匹配过程详解：**

第一步，计算代价矩阵。对每个 Query i 和每个真实物体 j，计算一个"代价"（cost），代价越低说明这个 Query 越适合负责这个物体。代价由三部分组成：

- 分类代价：Query 预测的类别和物体真实类别的交叉熵
- Mask 代价：Query 预测的 Mask 和物体真实 Mask 的 BCE + Dice
- 如果 Query 没有匹配到任何物体，它应该预测"无对象"类别

第二步，用匈牙利算法求解最优匹配。匈牙利算法保证找到"总代价最小"的配对方案，而且每个 Query 最多匹配一个物体，每个物体最多匹配一个 Query。

第三步，计算损失。只对匹配上的 Query-物体对算分类损失和 Mask 损失，没匹配上的 Query 算"无对象"损失。

**迁移路径：**

建议先用简化版跑通整个实验流程，确认模型能正常训练和收敛。然后再切换到完整版，对比两种损失函数的效果差异。

参考官方代码：`github.com/tue-mps/eomt/training/mask_classification_loss.py`
